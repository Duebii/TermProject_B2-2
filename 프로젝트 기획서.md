# 🤖 [기술 뉴스 자동 수집·요약 시스템]

## 1. 프로젝트 개요

### 프로젝트명

AI 기술 뉴스 자동 수집·요약 시스템

### 프로젝트 배경

새로운 프로젝트를 시작하거나 시장 조사를 수행할 때 최신 기술 동향을 파악하는 것은 매우 중요하다.

그러나 다양한 뉴스 사이트와 커뮤니티를 직접 방문하여 정보를 수집하고 정리하는 과정은 많은 시간과 노력이 필요하다.

특히 AI 산업은 기술 발전 속도가 빠르기 때문에 지속적인 정보 수집이 필요하지만, 반복적인 탐색과 정리는 생산성을 저하시킬 수 있다.

이에 본 프로젝트는 RSS 피드를 활용하여 최신 AI 기술 뉴스를 자동 수집하고, 생성형 AI를 통해 핵심 내용을 요약한 뒤, 노션 데이터베이스에 자동 저장하는 자동화 파이프라인을 구축하고자 하였다.

---

## 2. 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

- RSS 기반 뉴스 자동 수집
- 생성형 AI를 활용한 뉴스 요약 자동화
- 노션 데이터베이스 자동 저장
- 스케줄 기반 자동 실행
- 중복 저장 방지
- 예외 상황에 대한 에러 처리 적용
- 사람의 개입 없이 동작하는 자동화 시스템 구축

## 4. 사용 기술 및 선정 이유

### 자동화 도구

n8n과 Make를 함께 사용하는 ‘하이브리드 구조’로 설계했다.

### n8n 선정 이유

- 복잡한 데이터 가공 처리 가능
- Code Node를 통한 유연한 로직 구현
- Gemini API 연동에 적합
- 무료 환경에서도 충분한 활용 가능

### Make 선정 이유

- 외부 서비스 연결이 직관적
- Webhook 기반 연동이 안정적
- Notion 등 SaaS 서비스 연결이 편리함

### [하이브리드 구조] 선택 이유

n8n은 데이터 처리와 AI 요약을 담당하고, Make는 외부 서비스 연동 및 데이터 전송을 담당하도록 역할을 분리했다.

이를 통해 각 도구의 장점을 최대한 활용하면서 단점을 보완할 수 있었다.

#### 기대 효과

- 비용 최소화
- 학습 효과 극대화
- 실무형 자동화 구조 경험
- Webhook 기반 서비스 연동 경험 확보

## 5. 데이터 수집 전략

### RSS 뉴스 소스 선정

AI 관련 양질의 데이터를 확보하기 위해 국내외 AI 전문 매체를 선정하였다.

### 1) AI타임스

국내 대표 AI 전문 매체로 대부분의 콘텐츠가 인공지능 기술 및 산업 동향에 집중되어 있다.

일반 IT 기사나 제품 리뷰 등 프로젝트 목적과 무관한 데이터를 최소화할 수 있어 AI 전문 정보 수집에 적합하다고 판단하였다.

### 2) TechCrunch

글로벌 생성형 AI 스타트업 투자 동향 및 빅테크 기업의 신기술 발표 뉴스를 빠르게 제공하는 대표 기술 매체이다.

국내 시장뿐 아니라 글로벌 AI 산업 흐름을 함께 파악하기 위해 선정했다.

### 3) TheAI

AI 기술의 산업 적용 사례, 정책 및 규제 동향, 기업 비즈니스 모델 등을 심층적으로 다루는 전문 매체이다.

기술 정보뿐 아니라 실제 산업 활용 사례를 확보할 수 있다는 점에서 높은 활용 가치를 가진다고 판단했다.

## 8. 자동 저장 설계

### 저장 구조

수집된 뉴스 데이터는 Gemini API를 통해 요약된 후 Notion 데이터베이스에 자동 저장된다.

### 저장 항목

| 항목 | 설명 |
| --- | --- |
| 제목 | 기사 제목 |
| 요약문 | Gemini API 생성 요약 결과 |
| 원문 링크 | 기사 URL |
| 발행일 | 기사 발행일 |
| 출처 | RSS 뉴스 제공 매체 |

### 구현 방식

- n8n에서 뉴스 데이터 수집 및 AI 요약 수행
- Make를 통해 Notion 데이터베이스로 전달
- 각 속성에 맞게 자동 매핑 후 저장
- 저장 전 중복 여부 검증 수행

### 기대 효과

- 반복적인 데이터 정리 업무 제거
- 협업 환경에서 정보 공유 용이
- AI 요약 결과의 체계적 아카이빙 가능
- 데이터 무결성 확보

---

## 9. 중복 방지 정책

### 적용 방식

- 기사 GUID 및 URL 기준 중복 확인
- n8n에서 1차 중복 검사 수행
- Make의 Notion Search Objects 모듈을 활용한 2차 검증 수행
- 동일 기사 발견 시 저장 생략

### 선택 이유

네트워크 지연이나 동시 실행 상황에서도 동일 기사가 반복 저장되는 문제를 방지하고 데이터 무결성을 확보하기 위함이다.

---

## 10. 에러 처리 정책

자동화 시스템의 안정성과 데이터 무결성을 확보하기 위해 단계별 예외 처리 정책을 적용하였다.

### 1. AI API 호출 실패 대응

#### 적용 방식

- Gemini API 호출 실패 시 자동 재시도
- 3초 간격으로 최대 3회 재시도 수행
- 3회 모두 실패 시 워크플로우 종료

#### 선택 이유

일시적인 네트워크 오류나 API 응답 지연 상황에서 시스템이 즉시 중단되지 않도록 하고, 반복 실패 시 잘못된 데이터 저장을 방지하기 위함이다.

---

### 2. 중복 기사 저장 방지

#### 적용 방식

- n8n에서 GUID 기반 1차 중복 검사 수행
- Make에서 Notion Search Objects 기반 2차 중복 검사 수행
- 미존재 데이터만 저장 진행

#### 선택 이유

네트워크 지연 및 동시 처리 환경에서도 데이터 무결성을 확보하기 위함이다.

---

### 3. 불필요한 API 호출 방지

#### 적용 방식

수집된 뉴스가 모두 중복 기사일 경우 JavaScript Return([])을 활용하여 워크플로우를 조기 종료한다.

#### 선택 이유

불필요한 Gemini API 호출을 방지하여 비용을 절감하고 시스템 자원을 효율적으로 사용하기 위함이다.

---

### 4. 출처 데이터 누락 방지

#### 적용 방식

n8n 데이터 가공 단계에서 JavaScript 삼항 연산(Ternary Processing)을 활용하여 출처 정보를 검증 및 보정한다.

#### 선택 이유

데이터 전달 과정에서 발생할 수 있는 출처 정보 누락 문제를 방지하고 저장 데이터의 신뢰성을 확보하기 위함이다.
